Common Commit Types:

Here’s a breakdown of some common conventional commit prefixes:

feat: — For a new feature.

    : feat: add user authentication with OAuth2

fix: — For bug fixes.

    : fix: resolve login error when incorrect password is entered

chore: — For tasks like maintenance, refactoring, or other non-functional changes (e.g., updating dependencies, configuration changes).

    : chore: update dependencies to latest versions

docs: — For changes to documentation (README, inline code comments, etc.).

    : docs: update README with installation instructions

style: — For code formatting changes (whitespace, missing semicolons, etc.), without affecting the actual functionality.

    : style: fix indentation in utils.py

test: — For adding or modifying tests.

    : test: add unit tests for user model

perf: — For performance improvements.

    : perf: optimize image loading for faster rendering

refactor: — For code changes that neither fix bugs nor add features but improve the codebase in some way (e.g., refactoring functions, improving readability).

    : refactor: simplify user validation function

build: — For changes related to the build process (e.g., packaging, CI/CD configuration).

    : build: update webpack config for production builds

ci: — For changes to continuous integration configuration.

    : ci: add GitHub Actions workflow for deployment
